Das Programm Krypto verschlüsselt Dateien aller Art mit Hilfe einer geladenen oder
erzeugten Schlüsseldatei. Als Schlüsseldatei kann jeder Dateityp benutzt werden oder auch
gar keine Schlüsseldatei. Krypto benutzt eine mehrfache 256 Bit (Vollbit) Verschlüsselung
die rein auf Information der Schlüsseldatei basiert, das ist die technisch höchste
Verschlüsselungstiefe die überhaupt auf Computern möglich ist. Krypto kann weder von
Mensch oder Computern, ohne passende Schlüsseldatei oder Passwort unberechtigt
entschlüsselt werden.
Auch ist der Zugang zum Programm Krypto selbst per Benutzer Name und Benutzer Passwort
in höchster Verschlüsselungstiefe geschützt. Krypto gehört weltweit,
zu den sichersten Daten Verschlüsselungs Programmen überhaupt.
Es gibt der Zeit weltweit kein sicheres Daten Verschlüsselungs Programm als Krypto
welches zum Kauf angeboten wird.
Krypto ist absolut sicher vor unberechtigter Daten Entschlüsselung.

Warum ist 256 Bit die technisch höchste Verschlüsselungstiefe
die überhaupt auf Computern möglich ist ?

Ein Computer kennt immer nur 256 verschiedene Zeichen.
1 Zeichen = 1 Byte hat im Binär Zahlensystem genau 8 Bit.
1 Bit kennt nur die Schaltzustände: aus oder an bzw. 0 oder 1
Durch die Kombination dieser 8 Bit ergeben sich 256 Bit.
Die Berechnung dazu: 2 Schaltzustände hoch 8 Bit = 256 Bit
Diese 256 Bit sind adressiert im Dezimal Zahlensystem von 0 bis 255 = 256 Bit.
Computer arbeiten aber im Hexadezimal Zahlensystem.
Dort sind diese oben benannten 256 Bit adressiert von 00 bis FF = 256 Bit.
Ein Byte kann also garnicht unter Bit 0 oder über Bit 255 sein.
Deshalb ist 256 Bit die technisch höchste Verschlüsselungstiefe
die überhaupt auf Computern möglich ist.

Beweise der Krypto Sicherheit !
Was wäre, wenn man versuchen würde eine von Krypto verschlüsselte Datei

Eine verschlüsselte Datei mit der Länge von 18033 Zeichen hat demnach laut Berechnung,
256 Bit hoch 18033 Zeichen = 6,1843558143632013533192271736305e+43427 Datei Möglichkeiten.
Jede Datei Möglichkeit hat genau 18033 Zeichen bzw. Byte.
Das multipliziert mit der Anzahl Datei Möglichkeiten ergibt dann den Speicher Bedarf.
Das sind dann: 1,1152248840041161000440562362208e+43432 Byte.
Das sind dann: 1,0386341102459617890827881509632e+43423 Giga Byte Daten Menge.
Das ist eine Zahl mit 43424 Stellen.
Ich kann sicher behaupten soviel Speicher Platz gibt es auf der ganzen Welt nicht bzw. nie.

Wer sie nicht kennt kann ja da nur noch sagen. So genau weiß das keiner !
Sie können selbstverständlich natürlich auch noch nacheinander mehrfach verschlüsseln,
sogar bis zur Unendlichkeit, wobei dabei die Verschlüsselungstiefe dadurch nicht größer
werden kann. Mehr als 256 bit (Vollbit) geht nun mal nicht.

KRYPTO 2.0/2006 Professional Multi User
hat eine intelligente Krypto Benutzer Verwaltung eingebaut !
Damit können verschiedene Personen aber nicht alle ihr Krypto benutzen. Diese Krypto
Benutzer Verwaltung arbeitet mit selbst erzeugten verschlüsselten Dateien. Jede dieser
Dateien stellt einen Krypto Benutzer dar. Diese Dateien sind immer 80 Byte groß und sind
selbst bei gleichen Krypto Benutzer Eingaben immer komplett anders. Sie sind mit einer
256 Bit (Vollbit) Passwort Verschlüsselung sowie nachträglich auch noch per Zufall
verschlüsselt. Der Benutzer Name und das Benutzer Passwort müssen jeweils mindestens
6 Zeichen und maximal 50 Zeichen lang sein. Um neue Krypto Benutzer anzulegen müssen Sie
nur auf dem Krypto Feld doppelklicken mit linker Maustaste.

Es gibt zwei Sorten von Krypto Benutzern. Nur Krypto Master Benutzer können neue Krypto
Benutzer anlegen. Benutzer Name und Benutzer Passwort und Benutzer Datei Name werden in
eine Datei ihrer Wahl für Sie lesbar gelogt. Nur in dieser KRYPTO Benutzer LOG Datei
werden Benutzer Name und Benutzer Passwort gelogt. Passwörter die zur Daten
Verschlüsselung benutzt werden, sind grundsätzlich in Krypto nicht gelogt. Alle
Informationen zu diesem Thema sind bei der Bestellung von Krypto dabei. Ein
unberechtigtes Entschlüsseln dieser Krypto Benutzer Dateien kann ebenfalls absolut
ausgeschlossen werden. Krypto schreibt Log Dateien, dies ist dafür gedacht falls der
KRYPTO Benutzer viele Dateien mit verschiedenen Schlüsseln verschlüsselt, und später
nicht mehr weiß, welche Schlüsseldatei zu welcher verschlüsselten Datei gehört.
Krypto schreibt keine Log Dateien bei der Daten Entschlüsselung, da man ja nach der
Entschlüsselung nicht mehr wissen muss, wie vorher verschlüsselt wurde. Es kann auch
für viele Verschlüsselungen nur eine Log Datei benutzt werden. Das heißt Krypto hängt
neue Daten an schon vorhandener Log Datei an.

Auch bei der Schlüsseldatei Erstellung wird mit Log Dateien gearbeitet. Die Log Datei
können sie nachträglich verschlüsseln, verändern oder aber auch zerstören und löschen,
oder selbst lesen. Die Log Datei Erstellung ist nicht abschaltbar, da sich die Benutzung
der Log Datei für den Benutzer als fast unverzichtbar erwiesen hat. In der Log Datei
stehen nur benutzte Datei Namen mit Programm Kennung und Datum und Uhrzeit. Es hat also
nichts mit Spionage zu tun, sondern soll den Krypto Benutzer nur helfen seine Dateien
wieder zu entschlüsseln nach vielleicht längerer Zeit. Beim entschlüsseln wird der
Dateiname aus der verschlüsselten Datei ausgelesen und als Vorschlag zum Abspeichern
im Programm Krypto angezeigt. Dadurch wissen sie automatisch um was für eine Datei die
sie entschlüsseln wollen sich handelt, wenn sein muss sogar vor der Entschlüsselung der
verschlüsselten Datei. Der Datei Name der Original Datei in der verschlüsselten Datei
steht am Anfang in der verschlüsselten Datei ab dem zweiten Zeichen und ist die einzige
Information, die nicht verschlüsselt in der verschlüsselten Datei ist.
Dies ist absichtlich von mir so gemacht worden und stellt kein Sicherheits Risiko dar.

2.Absatz
------------------------------------------------------------------------------------------
Daten Passwort Verschlüsselung !
Die Original Datei die sie verschlüsseln möchten wird abgefragt und später eingelesen.
Ihr Passwort wird abgefragt und intern in Krypto verarbeitet. Die Passwort Länge muß
mindestens 6 Zeichen haben. Sie können aber auch über die Zwischenablage ganze Datei
Inhalte als Passwort in der Passwort Abfrage reinkopieren. Ich empfehle ihnen aber
grundsätzlich Passwörter mit mindestens 12 Zeichen zu benutzen. Die aus ihren Passwort
intern errechnete Schlüsseldatei wird nicht auf Festplatte gespeichert oder ausgelagert,
sondern bleibt nur bei der Arbeit von KRYPTO im Arbeitsspeicher (RAM), solange die intern
errechnete Schlüsseldatei von KRYPTO gebraucht wird. Die verschlüsselte Datei wird
erzeugt und abgespeichert. Danach werden die benutzen Datei Namen in einer von ihnen
erzeugten oder gewählten LOG Datei gespeichert mit Programm Kennung und Datum und Uhrzeit
wie es sonst auch in Krypto üblich ist.

Daten Passwort Entschlüsselung !
Die verschlüsselte Datei die sie entschlüsseln möchten wird abgefragt und später
eingelesen. Ihr Passwort wird abgefragt und intern in Krypto verarbeitet. Krypto kennt
ihr Passwort nicht, das heißt wenn ihr Passwort falsch ist, dann ist auch die Datei
Entschlüsselung falsch. Krypto zeigt ihnen nun immer den richtigen Dateinamen der
Original Datei als Vorschlag für Sie an. Nach übernehmen des Datei Namens der Original
Datei wird die entschlüsselte Datei nun erzeugt und abgespeichert. Diese erzeugte
Datei ist absolut identisch mit der Original Datei die mal mit Krypto verschlüsselt
wurde. Es werden keine LOG Daten erzeugt und abgespeichert.

Zusatzfunktion bei der Daten Passwort Verschlüsselung !
Wenn Sie im Krypto Eingabe Feld SCH eingeben und danach auf Datei Passwort
Verschlüsselung mit der linken Maustaste klicken, dann wird auch die errechnete
Passwort Schlüsseldatei auf der Festplatte zur späteren Verwendung zusammen mit der
verschlüsselten Datei gespeichert. Wenn Sie im Krypto Eingabe Feld SCH-KRY eingeben
und danach auf Datei Passwort Verschlüsselung mit der linken Maustaste klicken, dann
wird nur die errechnete Passwort Schlüsseldatei auf Festplatte ohne verschlüsselte
Datei gespeichert. Danach wird die Log-Datei gespeichert. Achtung: Diese Passwort
Schlüsseldateien sind nur mit der Datei Passwort Entschlüsselung kompatibel zu
benutzen. Sie können allerdings diese Passwort Schlüsseldateien auch mit der
anderen in Krypto eingebauten Verschlüsselung problemlos benutzen. Die dann erzeugte
Ver und Entschlüsselung ist aber nicht kompatibel mit der Datei Passwort Ver
und Entschlüsselung obwohl sie ganz ähnlich funktioniert. Wenn Sie im Krypto
Eingabe Feld VOSP eingeben und danach auf Datei Passwort Verschlüsselung mit der
linken Maustaste klicken, dann wird ohne Schlüssel oder Passwort nur mit dem
Internen Schlüssel in Krypto und per Zufall verschlüsselt. Dieser benutzte interne
Schlüssel ist nur für die Zusatzfunktion VOSP und EOSP in der Passwort Ver und
Entschlüsselung und nicht für die Zusatzprogramme VOSP und EOSP oder andere
Funktionen in Krypto zuständig. VOSP und EOSP als Zusatzfunktionen bei der
Datei Passwort Ver und Entschlüsselung in Krypto sind zur direkten Daten
Verschlüsselung gedacht, und nicht zum verschlüsseln von Dateischlüsseln.
Es wird hier von mir dringend angeraten die Zusatz Funktion VOSP und EOSP bei
der Datei Passwort Ver und Entschlüsselung nur in Ausnahme Fällen zu benutzen.
Denn diese Verschlüsselung kann ja von jedem Krypto Benutzer sofort entschlüsselt
werden ohne Schlüssel oder Passwort sofern er Zugang zu Krypto hat.

Zusatzfunktion bei der Daten Passwort Entschlüsselung !
Wenn Sie im Krypto Eingabe Feld SCH eingeben und danach auf Datei Passwort
Entschlüsselung mit der linken Maustaste klicken, dann wird die von ihnen vorher gewählte
Schlüsseldatei, die vorher mit der Datei Passwort Verschlüsselung erzeugt wurde, zur Datei
Entschlüsselung benutzt. Hierbei wird dann natürlich kein Passwort mehr benötigt. Wenn Sie
im Krypto Eingabe Feld EOSP eingeben und danach auf Datei Passwort Entschlüsselung mit der
linken Maustaste klicken, dann wird ohne Dateischlüssel oder Passwort nur mit den internen
Schlüssel in Krypto und per Zufall richtig entschlüsselt. Natürlich nur die Dateien die
vorher mit der Zusatz Funktion VOSP bei der Datei Passwort Verschlüsselung verschlüsselt
wurden.

So funktioniert die Daten Passwort Verschlüsselung !
1. Verschlüsselung: Aus ihrem Passwort wird intern eine Passwort Schlüsseldatei berechnet.
2. Verschlüsselung: Diese Schlüsseldatei wird mit einem Verschlüsselungs Algorithmus
                    welcher nur in Krypto ist, per Vollbit verschlüsselt.
3. Verschlüsselung: Danach wird diese Schlüsseldatei mit ein nur in Krypto intern
                    enthaltene geheime erechnete Passwort Datei Vollbit verschlüsselt.
                    Dies ist nicht die Passwort Schlüsseldatei aus der 1.Verschlüsselung.
4. Verschlüsselung: Danach wird die Original Datei mit der Schlüsseldatei Vollbit
                    verschlüsselt.
5. Verschlüsselung: Danach wird die verschlüsselte Datei nocheinmal, aber diesmal per
                    Zufall Vollbit verschlüsselt.
                    Erst jetzt wird die verschlüsselte Datei gespeichert.

Durch die zusätzliche Zufalls Verschlüsselung in der Daten Passwort Verschlüsselung wird
selbst bei gleicher Krypto Benutzung immer eine komplett andere Verschlüsselte Datei
errechnet. Diese Daten Verschlüsselung ist also eine fünffach (256 Bit) bzw. (Vollbit)
Verschlüsselung mit fünf komplett nacheinander unterschiedlich benutzten Verschlüsselungs
Verfahren. Ein unberechtigtes entschlüsseln dieser verschlüsselten Dateien kann ebenfalls
absolut ausgeschlossen werden für alle Zeiten und jede Technik.

3.Absatz
------------------------------------------------------------------------------------------
Daten Verschlüsselung !
Die Original Datei die sie verschlüsseln möchten wird abgefragt und später eingelesen.
Die Schlüsseldatei die sie zur Verschlüsselung der Original Datei gewählt haben wird
abgefragt und später eingelesen. Die verschlüsselte Datei wird erzeugt und abgespeichert.
Danach werden die benutzen Datei Namen in einer von ihnen erzeugten oder gewählten LOG
Datei gespeichert.

Daten Entschlüsselung !
Die verschlüsselte Datei die sie entschlüsseln möchten wird abgefragt und später
eingelesen. Die Schlüsseldatei die sie zur Entschlüsselung der verschlüsselten Datei
gewählt haben wird abgefragt und später eingelesen. Die entschlüsselte Datei wird erzeugt
und abgespeichert. Diese erzeugte Datei ist absolut identisch mit der Original Datei die
mal mit KRYPTO verschlüsselt wurde. Es werden keine LOG Daten erzeugt und abgespeichert.

So funktioniert die Daten Passwort Entschlüsselung !
1. Verschlüsselung: Ihre eingelesene Schlüsseldatei ist bereits schon verschlüsselt
                    berechnet.
2. Verschlüsselung: Diese Schlüsseldatei wird mit einem Verschlüsselungs Algorithmus
                    welcher nur in Krypto ist, per Vollbit entschlüsselt.
3. Verschlüsselung: Danach wird diese Schlüsseldatei mit ein nur in Krypto intern
                    enthaltene geheime erechnete Passwort Datei Vollbit entschlüsselt.
4. Verschlüsselung: Danach wird die verschlüsselte Datei nocheinmal, aber diesmal per
                    Zufall Vollbit entschlüsselt.
5. Verschlüsselung: Danach wird die Original Datei mit der Schlüsseldatei Vollbit
                    entschlüsselt. Erst jetzt wird die entschlüsselte Datei gespeichert.

Durch die zusätzliche Zufalls Verschlüsselung in der verschlüsselten Datei ändert sich
beim entschlüsseln die Reihenfolge gegenüber der Verschlüsselung, was ja ganz logisch ist.
Zuerst wird die Schlüsseldatei ausgerechnet aus der vorhandenen Schlüsseldatei oder
ihrem Passwort, dann wird die verschlüsselte Datei per Zufall entschlüsselt
(wie das funktioniert wird nicht verraten), und dann wird mit der intern errechneten
Schlüsseldatei die verschlüsselte Datei richtig entschlüsselt. Dies kann natürlich nur
funktionieren wenn alle Parameter die benötigt werden, absolut korrekt sind, ansonsten
gibt es schon bei kleinsten Veränderungen richtig unbrauchbaren Datenmüll.
Das ist auch der Grund warum es sehr schwierig ist so ein Programm wie Krypto überhaupt
zu programmieren. Sie sollten eigentlich schon hier deutlich erkennen, das Krypto in der
Computerwelt etwas ganz besonderes ist, und auch in Zukunft unschlagbar bleibt.
Die Gründe dafür liegen auf der Hand, denn höher als 256 Bit (Vollbit) kann niemand egal
auf welchen Computer verschlüsseln. Niemand braucht mehr Sicherheit als Krypto eingebaut
hat. Eine noch höhere bzw. mehrfache Vollbit Verschlüsselung als Krypto jetzt schon hat,
würde nur mehr Rechenzeit brauchen, und würde absolut nicht mehr Sicherheit bringen.
Die Bedienung von Krypto ist von mir schon sehr gut und verständlich konstruiert.
Hat man die Bedienung von Krypto ersteinmal verstanden braucht man überhaupt keine Krypto
Hilfe mehr, obwohl die Krypto Hilfe ja vorhanden ist.

4.Absatz
------------------------------------------------------------------------------------------
Schlüssel Daten Erzeugen !
Damit wird eine Zufalls Schlüsseldatei erzeugt die genau die Länge der Original Datei
hat die sie damit verschlüsseln möchten. Danach werden die benutzen Datei Namen in einer
von ihnen erzeugten oder gewählten LOG Datei gespeichert. Die erzeugte Schlüsseldatei
und LOG Datei kann auch noch öfters verwendet werden. Krypto ist auf kein Datenformat
festgelegt. Es ist also nur ein Vorschlag, das die Datei Endung (SCH) für Schlüssel
Dateien und (KRY) für verschlüsselte Dateien, und (VOS) für nochmal verschlüsselte
Schlüsseldateien und (LOG) für Log-Dateien steht. Krypto kann daher bei richtiger
Bedienung eigentlich nicht in Konflikt mit anderen Programmen kommen.

5.Absatz
------------------------------------------------------------------------------------------
Daten Zerstören !
Datei zerstören liest die Länge der zu zerstörenden Datei ein. Erzeugt Zufalls Daten und
überschreibt die zu zerstörende Datei damit bis maximal 3000 mal. Sie können die zu
zerstörende Datei frei einstellbar in Krypto mit bis zu 250 mal physikalisch auf ihrer
Festplatte mit Zufalls Daten überschreiben, bevor die zu zerstörende Datei dann entgültig
für immer zerstört wird. Dazu müssen sie nur im (KRYPTO Eingabe Feld)
eine Zahl von 1 bis 250 eingeben, und dann auf das Datei Zerstören Feld klicken.
Neu ist bei (KRYPTO 2.0/2006 Professional Multi User) das es bei über 250 mal dann in
fünfziger Schritten weitergeht bis maximal 3000 mal. Stehen andere Werte im
(KRYPTO Eingabe Feld), und sie klicken auf Datei Zerstören, dann wird die zu zerstörende
Datei nur einmal überschrieben und für immer endgültig zerstört. Würde man die mit KRYPTO
zerstörte Datei mit speziellen Programmen wieder herstellen, so würde man nur die mit
Zufallsdaten überschriebene zerstörte Datei wieder herstellen können, und diese wäre
natürlich immer absolut unbrauchbar. Und genau so soll es ja auch sein.

7.Absatz
------------------------------------------------------------------------------------------
Krypto hat noch fünf weitere Zusatz Programme und ein Hilfe Programm !
Diese Zusatz Programme können durch Tastatur Befehls Eingabe und einen Doppelklick der
linken Maustaste im (KRYPTO Eingabe Feld) gestartet werden. Krypto zeigt übrigens im
(KRYPTO Eingabe Feld)b bei der Arbeit an, an was Krypto gerade arbeitet.

Die Befehls Eingabe: ? oder doppelklick mit linker Maustaste aufs KRYPTO Eingabe Feld
zeigt ein Hilfe Fenster an, in dem die Formate der Krypto Zusatz Programme angezeigt
werden und die Zusatzfunktion bei der Datei Passwort Ver und Entschlüsselung.
Eine kleine Benutzer Hilfe für Krypto Anwender.

Die Befehls Eingabe: Z10500 erzeugt eine Zufalls Schlüsseldatei mit 10500 Byte. Sie können
                     den Byte Wert frei wählen. Damit wurden die öffentlichen
                     Schlüsseldateien erzeugt.

Die Befehls Eingabe: P10500:Peter erzeugt eine Passwort Schlüsseldatei mit 10500 Byte und
                     dem Passwort Peter. Diese wird verschlüsselt als Schlüsseldatei
                     abgespeichert. Sie können den Byte Wert frei wählen.
                     Das Passwort wird von ihnen ebenfalls frei gewählt und kann sehr
                     lang sein. Sie können sogar ins Passwort Eingabefeld über die
                     Zwischenablage Daten rein kopieren als Passwort, zb. ein Bild,
                     oder Musik oder was sie wollen.

Die Befehls Eingabe: OP:Peter erzeugt eine Passwort Schlüsseldatei mit der Länge der
                     Original Datei die sie verschlüsseln wollen, und dem Passwort Peter.
                     Diese Datei wird verschlüsselt als Schlüsseldatei abgespeichert. Das
                     Passwort wird von ihnen frei gewählt und kann sehr lang sein. Sie
                     können sogar ins Passwort Eingabefeld über die Zwischenablage Daten
                     rein kopieren als Passwort, zb. ein Bild, oder Musik oder
                     was sie wollen.

Die beiden Programme: (P10500:Peter) und (OP:Peter) benutzen ein mathematisches Verfahren
zur Daten Verschlüsselung, welches nur in Krypto verwendet wird und ein langes internes
Passwort in Krypto. Dieses verwendete lange interne Passwort von Krypto,
wird nur von den Zusatz Programmen (P und OP) benutzt.

Die Befehls Eingabe: VOSP verschlüsselt Dateien aller Art ohne Schlüssel und Passwort.
Dieses Programm benutzt ein mathematisches Verfahren welches nur in Krypto verwendet wird
und ein internes langes Passwort von Krypto. Dieses verwendete interne lange Passwort wird
nur von den Zusatz Programmen VOSP und EOSP benutzt. Das Programm VOSP ist dafür gedacht
um nachträglich eine verschlüsselte Passwort Schlüsseldatei nocheinmal zu verschlüsseln
mit dem internen Passwort von Krypto, aber als Schlüsseldatei für Krypto. Deshalb benutzt
dieses Programm eine eigene Dateiendung (VOS) die man benutzen kann, aber nicht muss.
Das interne Passwort von Krypto wird normaler weise nur in Verbindung mit ihrem vorher
selbst gewählten Passwort benutzt. Allerdings können sie auch jede andere Datei mit VOSP
verschlüsseln, in diesem Fall kann dann allerdings jeder Krypto Besitzer die
verschlüsselte Datei mit Krypto EOSP problemlos entschlüsseln. Dies sollten sie also
eigentlich aus Sicherheits Gründen absolut nicht tun. Denn VOSP ist eigentlich zum
verschlüsseln von Schlüsseln gedacht und nicht zur direkten Daten Verschlüsselung.

Die Befehls Eingabe: EOSP entschlüsselt Dateien die mit dem Programm VOSP verschlüsselt
wurden. Dabei wird jedoch der Original Dateiname nicht im Programm Krypto beim
entschlüsseln angezeigt. Das ist damit zu begründen das ja Schlüsseldateien auch für
mehrere Datei Verschlüsselungen eingesetzt werden können. Würde man aber eine
Schlüsseldatei verschlüsseln entsteht dadurch eine verschlüsselte Datei mit der Datei
Endung (KRY). Die Datei Endung (KRY) ist keine Schlüsseldatei, sondern eine verschlüsselte
Datei. Ein internes Passwort von Krypto, welches nur von den Zusatz Programmen
EOSP und VOSP benutzt wird in Verbindung mit ihrem frei gewählten Passwort ist geheim,
und kann nicht vom Kunden geändert werden. Die internen Passwörter von Krypto stellen
kein Sicherheits Risiko dar, weil man ja nach der Verschlüsselung mit Passwort und VOSP
die benutzte Schlüsseldatei, zerstören kann. Man kennt ja schließlich das Passwort und
das Zusatz Programm VOSP. Außerdem findet die eigentliche Verschlüsselung ja erst
nachträglich, mit der nachträglich mit VOSP erzeugten Schlüsseldatei statt. Es handelt
sich also hierbei sogar um eine dreifach Verschlüsselung, welche immer anders
funktioniert. Die letzte Verschlüsselung basiert wieder rein auf Information der
verwendeten Schlüsseldatei, und ist damit absolut sicher. Anhand der verschlüsselten
Datei ist es nicht möglich zu erkennen mit welcher Methode verschlüsselt wurde. Das
interne Passwort von Krypto wird nur in Verbindung mit ihrem Passwort bei der
Verschlüsselung mit Passwort vom Zusatz Programm VOSP bzw. EOSP benutzt. Es ist eine
